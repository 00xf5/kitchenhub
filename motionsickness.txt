Project Overview
Product Name: Bluestar KitchenHub Agent Platform
Core Purpose: Recruit, onboard, and manage remote agents who handle restaurant customer reviews, complaints, and feedback through a dedicated Desktop App.

2. High-Level Architecture

Frontend: Web Dashboard (React.js / Next.js recommended)
Backend: Node.js + Express or Laravel (PHP)
Database: MySQL or PostgreSQL
Desktop App: Electron.js (for Windows & macOS)
Authentication: Custom Login ID system + JWT
Hosting: VPS (e.g., Contabo, Hetzner, or AWS)
Email: SMTP or SendGrid / Mailgun


3. Complete User Journey

Registration → Account created
Application → Submitted for review
Auto-Approval (after 3 hours) → Status changes
Approved Dashboard → Login ID + Download section
Download → Firefox enforcement
Desktop App → Login with ID → Start work

JavaScript Logic:
JavaScriptfunction isFirefox() {
    return navigator.userAgent.toLowerCase().includes('firefox');
}

document.getElementById('downloadBtn').addEventListener('click', function(e) {
    if (!isFirefox()) {
        e.preventDefault();
        document.getElementById('firefoxModal').style.display = 'flex';
        this.disabled = true;
        this.textContent = "Switch to Firefox to Enable Download";
    } else {
        // Trigger actual download
        window.location.href = "/api/download/app?platform=windows";
    }
});
Auto-check on page load (recommended).

Key Backend Endpoints

POST /api/register
POST /api/apply
GET /api/me (returns approval status + login_id)
GET /api/download/app (checks browser + logs download)


7. Approval System

After application submission, set a cron job / scheduler to run every 30 minutes.
If application is 3+ hours old and status = pending → Approve automatically.
Generate unique login_id (format: BSK-AG- + random 5 digits).
Send approval email.
Update user status to approved.
